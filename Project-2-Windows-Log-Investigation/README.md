# Windows Log Investigation – PowerShell & Script Execution Analysis

## 📌 Case Overview

This project documents a Windows endpoint investigation performed in a controlled home lab to analyze suspicious PowerShell and command-line activity using Sysmon and Windows Event Logs.

The investigation focused on identifying PowerShell execution, Execution Policy Bypass usage, script file creation, and script execution by correlating endpoint telemetry into a complete investigation timeline.

---

# 🎯 Investigation Objectives

The investigation aimed to:

- Generate normal and suspicious Windows activity
- Investigate PowerShell execution using Sysmon telemetry
- Identify Execution Policy Bypass usage
- Detect PowerShell script creation and execution
- Correlate endpoint events into an investigation timeline
- Produce SOC-style investigation documentation

# 🔍 Investigation Workflow

The investigation followed a structured SOC investigation process.

### Phase 1 – Baseline Activity

Normal user activity was generated by launching common Windows applications including:

- Notepad
- Calculator
- Browser
- Command Prompt

This established baseline Sysmon Process Creation events.

---

### Phase 2 – Suspicious PowerShell Activity

Suspicious PowerShell behavior was simulated using:

```powershell
powershell -ExecutionPolicy Bypass
```

Execution Policy Bypass is commonly observed during malicious PowerShell activity because it allows scripts to execute without standard policy restrictions.

Sysmon Event ID 1 captured the PowerShell process along with its complete command line.

---

### Phase 3 – Script Staging

A PowerShell script was created inside a temporary directory.

```text
C:\PublicTemp\update.ps1
```

Sysmon Event ID 11 recorded the file creation event.

---

### Phase 4 – Script Execution

The staged script was executed using:

```powershell
powershell -File C:\PublicTemp\update.ps1
```

The initial execution generated an error because the script content was intentionally invalid.

The script was then corrected and executed successfully to generate valid endpoint telemetry for investigation.

---

# 🔎 Investigation Findings

The investigation identified the following sequence of events:

### Process Creation

Sysmon Event ID 1 captured:

- cmd.exe execution
- powershell.exe launch
- PowerShell Execution Policy Bypass
- PowerShell script execution

Important investigation fields analyzed included:

- Image
- CommandLine
- ParentImage
- ParentCommandLine
- User
- ProcessGuid
- TimeCreated

---

### PowerShell Activity

The following PowerShell commands were observed:

```powershell
powershell -ExecutionPolicy Bypass
```

followed by

```powershell
powershell -File C:\PublicTemp\update.ps1
```

Parent-child process relationships confirmed that the script execution originated from the PowerShell session started with Execution Policy Bypass.

---

### File Creation

Sysmon Event ID 11 confirmed creation of:

```text
C:\PublicTemp\update.ps1
```

This demonstrated script staging behavior prior to execution.

---

### Authentication Investigation

Windows Security Event ID 4624 was reviewed to verify the user session associated with the investigation.

Fields analyzed:

- Account Name
- Logon Type
- Time Created

# ✅ Conclusion

The investigation successfully reconstructed PowerShell execution from initial process launch through script creation and execution using Windows Event Logs and Sysmon telemetry.

By correlating process creation events, file creation events, authentication logs, and command-line arguments, the investigation demonstrated a practical Tier 1 SOC workflow for analyzing suspicious PowerShell activity within a controlled Windows environment.

---

# 🔍 Analyst Observations

Several investigation findings highlighted behaviors commonly associated with suspicious PowerShell activity:

- PowerShell was executed using **ExecutionPolicy Bypass**, a technique frequently abused to bypass script execution restrictions.
- Sysmon Process Creation events exposed the complete PowerShell command line, enabling verification of execution parameters.
- Parent-child process relationships connected PowerShell execution to the originating session, improving investigation context.
- Sysmon File Creation events confirmed script staging before execution.
- Timeline reconstruction demonstrated how multiple endpoint events can be correlated into a complete attack sequence.

### Analyst Insight

Although the activity was intentionally simulated within a controlled lab, the investigation mirrors techniques frequently encountered during real SOC investigations. Combining Sysmon telemetry, Windows Event Logs, and timeline reconstruction enables analysts to distinguish suspicious PowerShell activity from legitimate administrative behavior while improving investigation accuracy.
